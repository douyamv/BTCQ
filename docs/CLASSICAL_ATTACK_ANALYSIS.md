# 经典攻击成本分析与参数选择

本文件刻画 QXEB 在不同 (n, depth) 参数下经典攻击者需要付出的代价，以此驱动协议参数的选择与升级路径。

---

## 1. 三类经典模拟方法及其墙

### 1.1 状态向量精确模拟（最直观）

存 2ⁿ 个复数，每个 16 字节。

| 比特数 n | 状态向量大小 | 谁能跑 |
|---|---|---|
| 20 | 16 MB | 树莓派 |
| 25 | 512 MB | 老笔记本 |
| 28 | 4 GB | 任意现代笔记本 |
| 30 | 16 GB | 工作站、Mac Studio |
| 32 | 64 GB | 高端工作站 |
| 35 | 512 GB | 1 台带 1 TB 内存的服务器 |
| 38 | 4 TB | 小型集群 |
| 40 | 16 TB | 中型集群（数十节点） |
| 43 | 128 TB | 国家级超算（Summit 2019 跑过） |
| 45 | 512 TB | 极限：Frontier、Aurora、神威 |
| 50 | 16 PB | **任何超算都不够** |

**普通笔记本死线**: ~32 比特
**工作站死线**: ~38 比特
**全球最强超算死线**: ~46 比特

> **50 比特就是状态向量法的物理墙——再大整个地球的内存都不够。**

### 1.2 张量网络（近似 / 低纠缠精确）

不存完整状态，存"张量收缩图"。对浅电路特别有效。

| n × depth | 张量网络可行性 |
|---|---|
| 53 × 20（Sycamore） | 2019 Google 估 10⁴ 年 → 2022 Pan 等 15 小时（512 GPU） |
| 60 × 20 | 2024 顶级集群 几天到几周 |
| 70 × 20 | 几个月到几年，少数论文 demo 过 |
| 80 × 20 | 边界，需特殊结构、近似精度妥协 |
| 100 × 20 | **2026 现状：经典完全不可行** |
| 120 × 20 | 远超 |
| 156 × 20 | 天文数字 |

**张量网络真实死线（2026 年）**: ~80 比特 + 深度 20

注意：
- 深度越浅越容易（深度 5 的 100 比特经典可解）
- 拓扑越规则越容易（一维链最易，全连接最难）
- 2D 网格 + 深度 20 是 RCS 的"硬"区域

### 1.3 Schrödinger-Feynman 混合 / 切片法

把电路按比特切成两半各算，最后合并。

| n × depth | 切片法 |
|---|---|
| 50 × 20 | 几小时 |
| 60 × 20 | 几天 |
| 70 × 20 | 几周（边界） |
| 80 × 20 | 几年 |

**切片法真实死线**: ~70 比特 + 深度 20

### 3.1 验证瓶颈与缓解

随 n 增大，全节点验证的代价同步上升。我们的缓解策略：

* **n ≤ 38**：所有全节点用状态向量直接验证（v0.1–v0.2）
* **n ∈ [40, 55]**：每区块附带"哨兵子电路"（n_sentinel = 25），全节点只验证哨兵 + 主电路 XEB 的统计一致性。哨兵协议详见后续 PROTOCOL_v0.5.md
* **n ≥ 55**：引入零知识证明（ZK-SNARK）证明 XEB ≥ 阈值，全节点验证 SNARK，O(log) 时间。基于 Halo2 / Plonky2 实现
* **n ≥ 75**：增加量子委员会（提交者多签），双重保障

## 4. v0.1 工程妥协的诚实声明

v0.1 当前默认 n=30 的原因：

1. **可运行性优先**：让任何 16 GB Mac 都能在 30 秒内挖矿和验证
2. **教学价值**：协议机制完整呈现，规模可视为 v1.0 的等比缩小
3. **明确的弱点**：高端工作站（>= 64 GB RAM）能在分钟级伪造区块；这就是为什么 v0.1 是"测试网"而非主网
4. **升级路径预留**：n_qubits、depth、n_samples 都在 `qxeb/constants.py` 集中定义，硬分叉只需修改这些常量

如果你拥有强算力机器并希望体验"真量子优势"挖矿，可以本地修改这三个常量到 (40, 18, 8192) 自建 testnet，但需自行准备 16 TB 内存或张量网络验证器。

## 5. 更激进的目标：直接上量子比特数极限

理论上 QXEB 协议**没有 n 上限**——只受参与方硬件能力约束。156 比特的 IBM Heron r2 能挖 n=120 的电路（CZ 99.74% × 120×20 ≈ 0.94⁰·² 信号，需较强误差缓解）。Quantinuum H2 56 比特 + 99.87% 适合 n=50 高质量挖矿。

我们鼓励社区在条件成熟时分叉一条"激进链"，参数 ≥ (75, 20)，作为通往 v1.0 的实验场。
